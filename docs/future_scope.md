## 3. Ручное разбиение на треды
**Символ разрыва:** `///` (три слеша)

**Логика:**
- При встрече `///` в тексте — принудительно начинать новый блок
- Символ `///` не включается в финальный текст
- Работает вместе с автоматическим разбиением по 500 символов
- Приоритет: ручной разрыв > автоматический

**Пример:**
```
Первый пост о важной теме.
///
Второй пост с продолжением.
```
Результат: 2 отдельных блока, независимо от длины.

## 8. Механизм владельца (Owner Mode)

**Цель:** Разделить личные черновики от публичного инструмента

### Способ идентификации
**Специальная ссылка с токеном:**
```
https://site.com/?token=SECRET_TOKEN_HERE
```

**Логика:**
1. При переходе по ссылке с токеном:
   - Проверить токен на сервере
   - Создать cookie `owner_session` (срок: 30 дней)
   - Показать все черновики владельца

2. При обычном переходе (без токена):
   - Полный доступ к форматтеру
   - Черновики НЕ показываются
   - Кнопка "Drafts" скрыта

### Хранение
- Токен в `.env`: `OWNER_TOKEN=secret123`
- Cookie: HttpOnly, Secure, SameSite=Strict

### Что видит владелец
- Полный интерфейс
- Кнопка "Drafts" и sidebar
- Все сохранённые черновики

### Что видят другие
- Редактор и форматирование ✅
- Превью LinkedIn/Threads ✅
- Копирование ✅
- Черновики ❌ (кнопка скрыта)

### Безопасность
- Токен не отображается в UI
- Cookie удаляется при logout
- Возможность сменить токен в .env
